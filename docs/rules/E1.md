# E1: Text set in a font lacking its script's glyphs (silent fallback)

| | |
|---|---|
| Severity | error |
| Category | typography |
| Profiles | core, editorial, full |

## What it means

The font that will actually render this text has no glyphs for its script, so PowerPoint silently substitutes an OS fallback (Malgun Gothic for Hangul). No error is raised anywhere. Resolution follows the measured model: run a:ea > paragraph pPr/defRPr > lstStyle inheritance chain (shape > layout placeholder > master placeholder > master txStyles > defaultTextStyle) > theme ea (majorFont for title placeholders, minorFont otherwise; a non-empty theme ea beats run a:latin) > run a:latin only when the theme ea slot is empty > OS fallback.

## How to fix

Set a CJK-capable font on the run's a:ea slot. font.name (a:latin) alone only works when the theme ea slot is empty.
